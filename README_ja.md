# STFT Alignment Theory: Verification and Proposals for SciPy `ShortTimeFFT`

このリポジトリは、SciPyの信号処理モジュールにおける旧STFT実装から新実装（`ShortTimeFFT`）への移行に伴う**アライメント（時間軸の整列）の不透明性**を解消し、数学的に一貫した計算モデルを提案するためのプロジェクトです。

## 1. Motivation / 動機
SciPyのSTFT実装が従来の関数ベースからクラスベースの `ShortTimeFFT` へ刷新された際、デフォルト動作の変化や「目盛りの消失」といった課題が生じました。
本プロジェクトでは、実機検証（窓長 $M$、ホップサイズ $h$ の網羅的テスト）を通じて、設計思想のギャップを抽出し、理論的な再定義を行いました。

## 2. General Alignment Theory / アライメントの一般論
連続時間軸上での「アライメント」を、単なるインデックスの関係式ではなく、**物理時間軸上の一致条件**として数理的に定義します。

以下の3つの独立したパラメータ（自由度）を用いた**アライメント方程式**を導入します：
- $t[k]$ : 入力信号の時間軸
- $\tau[p]$ : 出力（STFT）の時間軸
- $k_{off}$ : サンプリング開始のオフセット

これにより、信号処理における「正しい時間軸管理」のフレームワークを提供します。

## 3. Analysis of Implementation / 実装の整理

### 旧STFTの仕様
旧実装が暗黙的に行っていたアライメントを上記の一般論で解釈します。
- 窓の中心を $m_{mid} = M // 2$ と扱ったり$M/2$と扱ったりしている一貫しない慣習について。
- `boundary` 引数に依存する $k_{off}$ の決定ロジックの明文化。

### 新STFT (`ShortTimeFFT`) の構造的制約
新実装における内部指標と制約を整理します。
- **8つの指標**: $p, k$ の最大・最小値および境界値（$p_{max}, p_{min}, p_{ub}, p_{lb}, k_{max}, k_{min}, k_{ub}, k_{lb}$）。
- **自由度の欠如**: 現状の実装では $\tau$ が $t[k_{off}]$ に強制的にロックされており、時間軸設定の柔軟性が制限されています。

## 4. Correspondence & Verification / 新旧の対応関係と検証
一般論に基づき、旧実装の整列パターンが8パターンに分類（窓の性質や境界条件の組み合わせ）、さらに終端のスライドの扱いも含めると24パターンになることを整理しました。一般論から見たときの新実装を整理し、新実装の整列の仕組みを明らかにしました。これにより、どんな時間軸を作成可能なのかが明らかになり、新旧実装を比較して「何が再現可能で、何が不可能なのか」を論じることができるようになりました。

- **論理的証明**: 特定の条件下で生じる $\Delta t/2$ （サンプリング間隔の半分）のオフセットが、現在の新仕様では表現不能であることを明らかにしています。

## 5. Improvement Proposals / SciPyへの改善要望
信号処理ライブラリとしてより堅牢で一貫性のある体系にするため、以下の2点を提案します。

1. **$\tau$ の独立化**: サンプリング間隔未満のオフセットを許容し、自由度を確保すること。
2. **一貫した計算体系の導入**: 現在 $k_{off}=0$ に依存している8指標の計算を、$t_{off}, \tau_{off}, k_{off}$ の3パラメータを基準とした汎用的なモデルへ移行すること。
